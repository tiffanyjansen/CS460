Step 3: Turn it into C# Code

This is the part that took the longest. The code itself wasn't too bad because Java is very similar to C#, but I had to remember to capitalize anything that was required to be capital according to C# conventions, and some of the other stuff was different than Java. For example, the Exception stuff. The Java code used super() and C# uses base() instead, they also use it in a very different way. (Seen to the right). This was the entire QueueUnderflowException class for each of the types. I was confused by some of the other differences between Java and C#, like the "for each" loop (Shown Directly Below). It was different enough that I had to Google the information about it to make sure that I did it the right way. The default() method threw me off too. In Java it's unneccesary, but when using C# it will give an error since we aren't sure what type of variable the "T" variable is. At first I thought I had to use the default method everywhere, but then I realized it was only in 1 specific spot that it was required. Other than that I found C# to be very similar to Java which made translating the application super simple. :)
